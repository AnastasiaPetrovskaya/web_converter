SchemaCrawler - Lint Example

Description

SchemaCrawler looks for potential database design issues. Follow the steps below to run SchemaCrawler Lint.

How to Run

  1. Follow the instructions in the commandline example
  2. To find schema design issues, run schemacrawler.cmd -server=hsqldb -database=schemacrawler -password= -infolevel=standard -command=lint (use schemacrawler.sh instead of schemacrawler.cmd on Unix)

How to Experiment

  1. See help on all of the available lints, run schemacrawler.cmd -command=lint -help (use schemacrawler.sh instead of schemacrawler.cmd on Unix)
  2. Create the SchemaCrawler Lint report in HTML5 and JSON formats.
  3. Create a customized SchemaCrawler Lint report using a linter configuration file.
  4. Copy the schemacrawler-linter-configs.xml file to the SchemaCrawler distribution directory, that is, the _schemacrawler directory.
  5. To find schema design issues, run with an additional command-line option, -linterconfigs=schemacrawler-linter-configs.xml
  6. Try creating your own database lints in a new jar - see SchemaCrawler Lint.
